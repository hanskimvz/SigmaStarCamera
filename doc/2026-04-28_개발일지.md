## 1. 해결한 문제들

### 1.1 config.json 누락 문제

#### 증상
```
CAM_FS_OK != s32Ret, open[/config/config.json] fail.
[MI ERR] [1302]: mi_common_CjsonGetCustomerCjson failed, config file is not exist
MDRV_FB_Probe[753] mi_fb init failed
```

#### 원인
`project/image/makefile`의 `gen_json` 타겟에서 `config.json`을 **무조건 삭제**하는 로직이 있었음:

```makefile
# 문제의 코드 (Line 184)
@rm -rf $(OUTPUT_JSON)
```

`CONFIG_JSON_MODULES_NUM=0`일 때 파일이 삭제만 되고 새로 생성되지 않음.

#### 해결 방법
`project/image/makefile` 수정:

```makefile
gen_json:
	@echo OUTPUT_JSON=$(OUTPUT_JSON)
	@echo CONFIG_JSON_MODULES=$(CONFIG_JSON_MODULES)
	@echo CONFIG_JSON_MODULES_NUM=$(CONFIG_JSON_MODULES_NUM)
	@if [ $(CONFIG_JSON_MODULES_NUM) -ne 0 ]; then \
		rm -rf $(OUTPUT_JSON); \
		echo { > $(OUTPUT_JSON);\
		# ... 기존 로직 ...
	else \
		echo "CONFIG_JSON_MODULES is empty, keeping existing config.json"; \
		if [ ! -f "$(OUTPUT_JSON)" ]; then \
			cp -f $(PROJ_ROOT)/../../etc/ref_board/config/config.json $(OUTPUT_JSON) 2>/dev/null || true; \
		fi; \
	fi;
```

### 1.2 VENC 펌웨어 버전 불일치

#### 증상
```
[ven-e][waves] fw ver:24120319 mismatch driver:23100720
[ven-e][vcodec] Failed to vpu init with bitcode, ret: 0x0000001e.
[ven-e][wrap] >>>>>>>>>>>>>>>>>>>>>venc load firmware fail<<<<<<<<<<<<<<<<<<
[ST_VencModuleInit 1499]exec function failed, error:a0022007
```

#### 원인
| 구성요소 | 버전 | 파일 |
|----------|------|------|
| SDK의 VENC 펌웨어 | 24120319 | `chagall.bin` (259,904 bytes) |
| SDK의 VENC 드라이버 | 23100720 | `mi_all.ko` 내부 |

펌웨어와 드라이버 버전이 맞지 않음.

#### 해결 방법
레퍼런스 보드의 `chagall.bin` (253,376 bytes)으로 교체:

```bash
# 레퍼런스 보드에서 추출한 파일로 교체
cp etc/ref_board/config/venc_fw/chagall.bin \
   Maruko-ILS00_TINY_V1.5.0/project/board/i6c/venc_fw/chagall.bin
```
