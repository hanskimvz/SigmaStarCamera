## 2. 파티션 구조 변경: /customer → /user

### 2.1 변경 목적
- 기존 `/customer` 파티션을 `/user`로 변경
- 커스텀 드라이버, 앱, 라이브러리 배치용

### 2.2 수정 파일

#### `project/image/configs/general/nor.squashfs.partition.config`
```diff
-IMAGE_LIST = boot kernel rootfs miservice customer
+IMAGE_LIST = boot kernel rootfs miservice user

-customer$(RESOURCE) = $(OUTPUTDIR)/customer
-customer$(MOUNTTG)  = /customer
+user$(RESOURCE)     = $(OUTPUTDIR)/user
+user$(MOUNTTG)      = /user
```

#### `project/image/configs/general/user.mk` (신규 생성)
- `customer.mk` 기반으로 작성
- `/user` 파티션 구조:
  ```
  /user/
  ├── drivers/    # 커스텀 드라이버 (.ko)
  ├── apps/       # 애플리케이션
  ├── lib/        # 공유 라이브러리
  └── demo.sh     # 초기화 스크립트
  ```

#### `project/image/configs/general/rootfs.mk`
```diff
-/customer/demo.sh → /user/demo.sh
-/customer/ssh     → /user/ssh
```

## 3. 디스플레이 모듈 비활성화

### 3.1 목적
- 카메라 전용 펌웨어로 디스플레이 기능 불필요
- 플래시 용량 절약

### 3.2 defconfig 수정
`project/configs/defconfigs/ipc_i6c.nor.uclibc-9.1.0-squashfs.027d.256.qfn128_defconfig`:
```diff
-CONFIG_INTERFACE_DISP=y
-CONFIG_interface_disp="enable"
+CONFIG_INTERFACE_DISP=y
+CONFIG_interface_disp="disable"

-CONFIG_INTERFACE_FB=y
-CONFIG_interface_fb="enable"
+CONFIG_INTERFACE_FB=y
+CONFIG_interface_fb="disable"

-CONFIG_INTERFACE_PANEL=y
-CONFIG_interface_panel="enable"
+CONFIG_INTERFACE_PANEL=y
+CONFIG_interface_panel="disable"
```

### 3.3 빌드 시스템 수정

#### `miservice.mk` - 모듈 복사 시 조건부 스킵
```makefile
for line in `cat $(LIB_DIR_PATH)/modules/$(KERNEL_VERSION)/.mods_depend`; do
    skip=0
    if [ "$(interface_disp)" = "disable" ] && [ "$${line}" = "mi_disp" ]; then skip=1; fi
    if [ "$(interface_fb)" = "disable" ] && [ "$${line}" = "mi_fb" ]; then skip=1; fi
    if [ "$(interface_panel)" = "disable" ] && [ "$${line}" = "mi_panel" ]; then skip=1; fi
    # ... 복사 로직
done
```

#### `user.mk` - demo.sh 생성 시 조건부 insmod 스킵
동일한 로직으로 비활성화된 모듈의 insmod 명령 생략

## 4. mi_all.ko 호환성 문제

### 4.1 시도
- `mi_all.ko` (단일 통합 모듈)를 사용하여 모듈 로딩 단순화 시도

### 4.2 실패 원인
```
insmod: can't insert '/config/modules/5.10/mi_all.ko': unknown symbol in module or invalid parameter
```
- V1.5.0/V1.3.0의 `mi_all.ko`는 다른 커널 버전용
- **vermagic 불일치** 문제

### 4.3 해결
- 개별 `mi_*.ko` 모듈 로딩 방식으로 복귀
- `.mods_depend` 파일 기반 의존성 순서대로 로딩

---

## 5. config.json 제거

### 5.1 용도
- `config.json`: FB/DISP 모듈의 하드웨어 설정 (해상도, 타이밍 등)
- `modparam.json`: MI 모듈 파라미터 설정

### 5.2 수정
- 디스플레이 모듈 비활성화 시 `config.json` 불필요
- `image/makefile`의 `gen_json` 타겟 조건부 실행:
```makefile
if [ $(CONFIG_JSON_MODULES_NUM) -ne 0 ]; then
    # JSON 파싱 실행
fi
```

## 6. mixer 애플리케이션 설정 문제

### 6.1 문제
```
Could not find the FILE[]
[ERR]:[read_config_file][2009]: ERROR read file !
```

### 6.2 원인
- JSON 설정 파일에 `/customer/` 경로 하드코딩
- `/customer` → `/user` 변경 후 경로 불일치

### 6.3 해결: user.mk에 자동 경로 치환 추가
```makefile
# mixer 복사 후 경로 치환
find $(user$(RESOURCE))/mixer -name "*.json" -exec sed -i 's|/customer/|/user/|g' {} \;
find $(user$(RESOURCE))/mixer -name "*.json" -exec sed -i 's|/user/iq/iq_day.bin|/config/iqfile/iqfile0.bin|g' {} \;
find $(user$(RESOURCE))/mixer -name "*.json" -exec sed -i 's|/mnt/I6C/SC430AI_MIPI_api.bin|/config/iqfile/iqfile0.bin|g' {} \;
```
